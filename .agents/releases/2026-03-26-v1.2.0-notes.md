# codereview v1.2.0

## Highlights

Large diffs no longer overwhelm the review. Diffs over 80 files or 8000 lines are automatically split into risk-tiered chunks, each reviewed independently, then synthesized into a single verdict. No configuration needed — it just works.

Spec verification now checks behavior, not just existence. Previously, a function with the right name and some tests could pass verification. Now the reviewer traces decision rules, parameter constraints, and edge cases to confirm the implementation actually matches the spec.

## What's New

- **Chunked review for large diffs** — risk-tiered splitting, cross-chunk analysis, and directory-based clustering keep reviews thorough even on massive changesets. Control with `--no-chunk` and `--force-chunk` flags, or tune thresholds in `.codereview.yaml`.
- **Behavioral spec verification** — the spec-verification pass and judge now verify that code behavior matches requirements, catching the common trap of "function exists and has tests" without correct logic.
- **Deeper correctness analysis** — new phases detect nil/partial objects on skip paths and type mismatches across serialization boundaries (e.g., array marshalled but map expected on the other side).
- **Deeper security analysis** — new phases trace inter-component data flows into dangerous sinks and detect environment variable namespace hijacking from dynamic sources.
- **Local CI release gate** — `ci-local-release.sh` validates skill integrity, schema-prompt consistency, shell script quality, and absence of secrets before every release.
