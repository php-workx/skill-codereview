# v1.3.0 — Deterministic Pipeline + Ruby/Java Support

## Highlights

This release replaces agent-interpreted review steps with a deterministic scan pipeline — 9 executable scripts that run 11+ linters and scanners in parallel with reproducible JSON output. Reviews are now faster, more consistent, and easier to debug.

Ruby and Java join Python, Go, Rust, and TypeScript as fully supported languages, with linting, security scanning, coverage collection, complexity analysis, and project discovery for each.

## What's New

- **Deterministic scan pipeline** — semgrep, trivy, osv-scanner, gitleaks, shellcheck, clippy, ruff, golangci-lint, eslint, rubocop, brakeman, and pmd run automatically with per-tool timeouts, parallel execution, and normalized JSON output
- **Ruby and Java at full parity** — RuboCop and PMD for linting, Brakeman for Rails security, SimpleCov and JaCoCo for coverage, flog for complexity, Gemfile/Maven/Gradle for project discovery
- **Finding lifecycle** — findings are fingerprinted across reviews so recurring issues are flagged automatically; suppress or defer findings with expiry dates and scope controls
- **Four-expert review panel** — the judge now reasons as Gatekeeper, Verifier, Calibrator, then Synthesizer for more structured and rigorous analysis
- **Git risk and coverage context** — reviewers see which files have high churn rates and low test coverage before diving into the code
- **Smarter detection** — new correctness patterns catch empty-parameter edge cases, classification rule gaps, multi-run state bugs, and routing table completeness issues; new security patterns catch command injection from file-derived input
