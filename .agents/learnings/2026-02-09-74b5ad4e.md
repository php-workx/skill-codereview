# Learning: Canonical prompt file location matters

**ID**: L1
**Category**: architecture
**Confidence**: high

## What We Learned

Root `prompts/reviewer-*.md` files were unused mirrors of `skill/prompts/` — the install script copies the entire `skill/` directory, making only `skill/prompts/` the canonical location. The only root prompt that matters is `prompts/codereview.md` (Codex dispatcher).

## Why It Matters

Maintaining unnecessary mirrors creates confusion and drift risk; identifying the single source of truth prevents wasted effort syncing files that nothing references.

# Learning: Schema backward compatibility via derivation

**ID**: L2
**Category**: architecture
**Confidence**: high

## What We Learned

When replacing a flat output field (`spec_gaps: string[]`) with a richer structure (`spec_requirements` array with per-requirement traceability), keep the old field populated by deriving it from the new structure in the judge step. This maintains backward compatibility without code duplication.

## Why It Matters

Consumers of the review JSON output won't break when the schema evolves, and the derivation is a single step in the judge rather than maintaining two parallel data flows.
